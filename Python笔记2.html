通过dirt的方法查找值，如果key不在，就会报错，避免key不在会报错，有两种方法：
1.通过in 判断key是否存在：
key值 in dirt
如果key 存在，就会返回true,如果不存在，就返回false.
2.通过get()方法，如果key不存在e已返回none,或者自己指定value。
dirt.get('throm')
如果不存在就返回None,但是在python命令行中不显示结果。
要删除一个key，用pop（key）方法,对应的vlaue也会从dirt中删除。
dirt.pop('throm')
这样throm和她的vlaue 都会被删除。
3.和list的比较：
dirt:
a.和list比，dirt查找的速度更快，不会随着key的增多变慢。
b.单dirt占用的内存较多。
list：
a.查找和插入的时间会随着元素的增加而增加；
b.占用内存少，内存浪费少。
4.需要注意一点：dirt的key是不可变的!!!通过key来找元素，如果总是变化，那么dirt会乱套的。这个通过key找value的方法叫做哈希算法。在Python中，字符串，整数都是不可变的，因此可以放心的作为key,但是list不可以，因为list是可变的。

set

set是一组key的集合，但是不存储value。由于key不能重复，所以在set中没有重复的key。
1.创建：
set(['a','b','c']) 回车
{'a','b','c'}
显示'a','b','c'不代表set是有序的，只是说set中有'a','b','c'这三个元素。
2.重复元素在set中会被过滤
set(['a','b','c','a'])回车
{'a','b','c'}
3.add(key)方法添加元素到set中，但是重复添加不会有效果。
4.remove(key)方法删除元素。
5.和dirt的区别：
区别：set没有没有对应的vlaue值，也不可设置。
相同点：set和dirt一样，不可以放入可变元素。，因为无法判断可变元素是否相等，无法保证set内部”无重复元素“的规则

不可变对象

str是不可变对象，list是可变对象。
对于可变对象：

a = ['b','c','a']
a.sort()
a = ['a','b','c']
对于不可变对象：

a = 'abc'
a.replace('a','A')
a = 'abc'

a是变量，而字符串‘123’才是字符串对象，a指向的是内容是字符串‘abc’,当我们调用a.relpace()方法的时候，实际上这个方法是作用在字符串‘abc’上的。不改变原来的字符串，会返回修改后的新的字符串。

结论：
对于不可变对象来说，调用自身的方法，不会改变自身，但是会返回一个新的改变后的值，可以用一个变量接收。
