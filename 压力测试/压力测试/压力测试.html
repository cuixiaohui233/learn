压力测试

1.安装 jmeter

brew install jmeter --with-plugins

2.安装 Javasdk，并配置环境变量。(必须为1.8及以前版本，目前jmeter不支持1.9)

I.去官方网站下载
II.配置环境变量
首先确认本机所使用的shell是哪个：zsh/bash
在命令行中输入
echo $SHELL ，
如果输出 /bin/bash 则为 bash，
如果输出结果为 /bin/zsh 则为 zsh.

根据上面不同的结果修改不同的shell配置文件
若为bash，则打开 vim ~/.bash_profile，若为zsh打开 vim~/.zshrc。

在相应的文件末尾添加以下内容，并保存
export JAVA_HOME=$(/usr/libexec/java_home)

在~/目录，命令行执行source命令
source .bash_profile 或 source .zshrc。

如果切换Java的版本，输入一下命令

  • 切换指定版本
export JAVA_HOME=`/usr/libexec/java_home -v 1.6.0_65-b14-462`
  • 切换主要版本
export JAVA_HOME=`/usr/libexec/java_home -v 1.8`

3.重新运行meter

open /usr/local/bin/jmeter

4.新建一个线程组

右击测试计划->添加->threads->线程组->设置线程数->

5.新建HTTP代理服务器

右击工作台—>添加非测试元件->HTTP代理服务器
端口：默认8888

6.谷歌浏览器添加 Proxy SwitchyOmega 插件

谷歌添加扩展程序：Proxy SwitchyOmega

7.配置prxy

代理协议：HTTP，
代理服务器：localhost,
代理端口：8888（默认）,

8.回到 jmeter，开始录制

点击start->选择目标目标控制器

9.清空浏览器的缓存

Appliclition->clear site data

10.开始录制

走测试流程，执行要测试的动作

11.选取关注的报告

我们关注的：

  • Graph Results 图形报告
  • Summary Report 聚合报告

12.执行测试

点击绿色开始按钮

术语与解释

（1）吞吐率（Requests per second）
概念：服务器并发处理能力的量化描述，单位是reqs/s，指的是某个并发用户数下单位时间内处理的请求数。某个并发用户数下单位时间内能处理的最大请求数，称之为最大吞吐率。
计算公式：总请求数 / 处理完成这些请求数所花费的时间，即Request per second = Complete requests / Time taken for tests
（2）并发连接数（The number of concurrent connections）
概念：某个时刻服务器所接受的请求数目，简单的讲，就是一个会话。
（3）并发用户数（The number of concurrent users，Concurrency Level）
概念：要注意区分这个概念和并发连接数之间的区别，一个用户可能同时会产生多个会话，也即连接数。
（4）用户平均请求等待时间（Time per request）
计算公式：处理完成所有请求数所花费的时间/ （总请求数 / 并发用户数），即Time per request = Time taken for tests /（ Complete requests / Concurrency Level）
（5）服务器平均请求等待时间（Time per request: across all concurrent requests）
计算公式：处理完成所有请求数所花费的时间 / 总请求数，即Time taken for / testsComplete requests
可以看到，它是吞吐率的倒数。
同时，它也=用户平均请求等待时间/并发用户数，即Time per request / Concurrency Level
